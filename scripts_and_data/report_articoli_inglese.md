## Proposta gestione lingua

**Opzione A – Cartella `src/content/blog/en/`**
- Spostare i file .md in inglese in una sottocartella `en/` (es. `src/content/blog/en/breaking-barriers-my-journey-as-an-autistic-actor.md`).
- Pro: URL e routing chiari (es. `/blog/en/...`), facile filtrare in build.
- Contro: richiede aggiornare link, redirect e generazione pagine (Astro) per due alberi (it/en).

**Opzione B – Tag `lang: en` nel frontmatter**
- Aggiungere nei file rilevati: `lang: en` (e eventualmente `lang: it` negli altri).
- Pro: nessuno spostamento file; filtri e layout possono usare `lang` (es. `if (lang === 'en')`), i18n in un solo albero.
- Contro: URL resta uguale; per path tipo `/en/...` serve routing via query o segmento derivato da `lang`.

**Raccomandazione:** **Opzione B** (tag `lang: en`) per ora: nessuna ristrutturazione cartelle, gestione via codice e filtri. Se in futuro servisse sito bilingue con path separati, si può migrare a Opzione A.
